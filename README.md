# Задание 1

### Текст задачи
Разработать аннотацию @Invoke, которая применяется только к методам, доступна во время
выполнения программы и не имеет свойств.  
Создать класс с методами, помеченными данной аннотацией, и реализовать обработчик,
который автоматически вызывает такие методы с использованием Reflection API.

### Алгоритм решения
1. Создать аннотацию @Invoke с целевым элементом METHOD.
2. Установить политику хранения аннотации RUNTIME.
3. Создать класс с несколькими методами, один из которых помечен @Invoke.
4. С помощью Reflection API получить список методов класса.
5. Проверить наличие аннотации @Invoke.
6. Автоматически вызвать помеченные методы.

## Задача 2 — Аннотация @Default

### Текст задачи
Разработать аннотацию @Default, которая может применяться к классу или полю,
доступна во время выполнения программы и содержит обязательное свойство value типа Class.
Реализовать обработчик, выводящий имя класса по умолчанию.

### Алгоритм решения
1. Создать аннотацию @Default с целями TYPE и FIELD.
2. Добавить обязательное свойство value типа Class.
3. Проаннотировать демонстрационный класс.
4. С помощью Reflection API получить аннотацию.
5. Вывести имя класса, указанного в свойстве value.

## Задача 3 — Аннотация @ToString

### Текст задачи
Разработать аннотацию @ToString, которая управляет формированием строкового
представления объекта.  
Аннотация должна иметь необязательное свойство со значениями YES или NO.

### Алгоритм решения
1. Создать перечисление Mode со значениями YES и NO.
2. Создать аннотацию @ToString с параметром по умолчанию YES.
3. Проаннотировать класс и одно из его полей.
4. С помощью Reflection API получить все поля класса.
5. Исключить поля с @ToString(Mode.NO) из строкового представления.
6. Сформировать итоговую строку объекта.

## Задача 4 — Аннотация @Validate

### Текст задачи
Разработать аннотацию @Validate, содержащую массив классов для проверки.
Аннотация должна применяться к типам и быть доступной во время выполнения программы.

### Алгоритм решения
1. Создать аннотацию @Validate с целями TYPE и ANNOTATION_TYPE.
2. Добавить обязательное свойство типа Class[].
3. Проаннотировать демонстрационный класс.
4. С помощью Reflection API извлечь массив классов.
5. Вывести список классов для проверки.

определяющий кешируемые области.
